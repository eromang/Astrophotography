## Framing — M16 + M17 in one RedCat field

Both nebulae fit in a **single [[RedCat-51]] frame** (2.45° apart, inside the 5.38°×3.60° FOV). Same filter, same subs — every 300 s exposure integrates **both** simultaneously, so the 135 min logged against each target is *shared frames*, not separate capture time.

| Parameter | Value |
|-----------|-------|
| Frame centre | RA 18h 19.8m, Dec −15.0° (M16↔M17 midpoint) |
| M16 → M17 separation | 2.45° (M17 is SSE of M16) |
| **Long-axis sky position angle** | **169°** (M16 / north end tipped ~11° toward W) |
| Stellarium Oculars *Rotation* field | **79°** (its zero is the RA axis: 90° + 79° = 169°) |
| Long-axis margin each end | ~1.4–1.5° clear |
| Sensor / scope | [[ASI2600MCPro]] (IMX571) + [[RedCat-51]] — verified in Oculars |

**On the rig (manual rotation — no CAA):** rotate by hand and **plate-solve in ASIAIR**, adjusting until the reported angle ≈ **169°** sky PA (or its 90° complement, per ASIAIR's axis convention). Match the framing, not the raw number — invariant: **long axis through both nebulae, M16 end tipped ~11° west.** Lock the rotator hard before slewing.

> [!tip] ASIAIR GoTo: **18h 19.8m / −15° 00′ @ PA 169°**
> Point at the **midpoint**, not M16 — centring on M16 clips M17's edge (2.45° sep vs 2.69° half-frame). Use the **Framing & Mosaic** tool: search M16, recentre to these coordinates, set Image Rotation 169°, GoTo. Then plate-solve and confirm centre ≈ 18h 19.8m / −15° and angle ≈ 169° before starting the sequence.
